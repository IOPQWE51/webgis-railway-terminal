## 🎥 NEW v2.0：智能摄影决策系统 (AI Photography Engine)

### **V3.0 卫星网关演算引擎** ⭐
这是 v2.0 最重要的新增模块，将您的每一个拍摄地点转化为"摄影智库"。

#### 🎯 核心功能
```
任意位置点击 → 启动 V3.0 卫星网关 → 实时调用多维环境数据 → 生成综合摄影评分 → 推荐最佳拍摄策略
```

#### 📊 评分维度与算法

| 维度 | 计算方法 | 权重 | 触发条件 |
|------|--------|------|---------|
| **黄金/蓝调时刻** | 日出日落计算 | +25/+20 | `[(当前时间) vs (SunCalc推算时刻)]` |
| **阴天柔光** | 云量判断 | +15 | 云量 > 70% 且无降水 |
| **浓雾/晓雾** | 湿度 + 能见度 | +25 | 湿度 ≥ 90% 且能见度 < 2000m |
| **降雪/雨景** | 天气类型 | +20/+15 | 检测 Snow/Rain 预报 |
| **绝佳观星** | 月相 + 云量 + 时间 | +15 | 月相 < 20% + 云量 < 20% + 夜间 |
| **海上满月** | 月相 + 地形 | +10 | 月相 > 80% + 海岸地点 |
| **决定性瞬间** | 规则引擎 | 动态 | 见下表 |

#### 🧠 决定性瞬间智能预警系统

系统通过配置化规则触发上下文智能建议：

```javascript
例 1: 赛博朋克触发 (Rain + 车站/圣地)
输出：🌧️ "寻找路面积水，利用车站/路灯霓虹灯拍摄高反差倒影！"

例 2: 圣地白雪触发 (Snow + 动漫取景地)
输出：❄️ "二次元取景地遇上降雪，日系极简高调大片诞生！"

例 3: 晨雾云海预警 (湿度 ≥ 90% + 清晨)
输出：☁️ "高湿度叠加清晨低温，极大概率出现平流雾或局部云海！"

例 4: 火烧云预警 (云量 30-70% + 黄金时刻)
输出：🔥 "完美云量叠加黄金时刻，准备广角镜头迎接漫天红霞！"
```

#### 🎯 评级系统
```
S 级 (≥ 90) ⭐⭐⭐⭐⭐ → 绝佳决定性瞬间 (强烈推荐)
A 级 (≥ 75) ⭐⭐⭐⭐   → 出片率极高 (条件优越)
B 级 (≥ 60) ⭐⭐⭐     → 适合记录 (光影普通)
C 级 (< 60)  ⭐⭐      → 不推荐强求 (建议现场踩点)
```

#### 📍 光影坐标与时间窗口
点击任何标记，系统会自动计算该位置的：
- 🌅 日出/日落时刻（当地时间）
- ✨ 黄金时刻（日落前 1 小时）
- 🌌 蓝调时刻（日落后 40 分钟）
- 🌕 月升/月落（若当月）

所有时间会根据**经度自动转换为当地时区**。

#### 🚀 触发方式
1. 地图上任意点击标记 → 打开 Popup
2. 点击 Popup 中的 **"🔮 启动 V3.0 卫星网关演算"** 按钮
3. 系统后台调用：
   - 📡 `SunCalc.getTimes()` - 太阳/月相位置
   - 🌐 `WeatherAPI` - 实时气象数据
   - 🗺️ `Overpass API` - 地形地貌特征
   - 📋 `decisiveMomentRules[]` - 规则库匹配
4. 1~3 秒后显示全面评估结果

- ⚠️ **登机口地狱级称重**：超重费巨高，无任何人情商量

**乐桃航空 (MM)**
- 🚩 **航站楼迷宫**：大阪关西 (KIX) 独占 T2，需乘接驳巴士 10 分钟
- ⚠️ **误机风险高**：务必比其他航班多预留 40 分钟

## 🎯 核心交互流程 (Interaction Flows)

### **工作流 #1: 使用 V3.0 卫星网关评估摄影位置** ⭐ NEW
```
用户在地图上点击任意标记 (🚉 车站/🌸 圣地/📍 地标)
    ↓
Popup 打开，显示基本信息 (名称/描述/当地时间)
    ↓ [用户点击 "🔮 启动 V3.0 卫星网关演算"]
┌─────────────────────────────────────────┐
│ ⚡ 后台并行调用多个 API                 │
│ 1. SunCalc.getTimes() - 太阳位置       │
│ 2. WeatherAPI - 气象数据               │
│ 3. Overpass API - 地形特征             │
│ 4. decisiveMomentRules[] - 规则匹配   │
└─────────────────────────────────────────┘
    ↓ [1~3秒后]
系统生成评分结果
    ├─ 综合摄影指数 (S/A/B/C 级)
    ├─ 具体评分 (0~100)
    ├─ 评语 + 拍摄建议
    ├─ 决定性瞬间预警 (若触发)
    ├─ 当地光影坐标
    │   ├─ 日出时刻
    │   ├─ 黄金时刻 (日落前 1h)
    │   ├─ 蓝调时刻 (日落后 40m)
    │   └─ 日落时刻
    └─ 导航按钮 (Google/Yahoo!)
```

```
用户输入地点名称
    ↓

```

    ↓

    ↓

```

    ↓

    ├─ 地图继续缩小至所有点保留在视图内
    └─ 重复直到所有点完全分散

    ↓ [用户点击单个标记]

```

## 🐛 常见问题与故障排查

### Q1: 启动 V3.0 卫星网关时提示"数据链路断开"
**原因**：WeatherAPI 或 Overpass API 请求失败
**解决**：
- 检查网络连接
- 验证 `VITE_WEATHER_API_KEY` 是否正确配置
- 查看浏览器控制台 (F12) 中的具体错误信息
- 可能是 API 配额已满 → 等待配额重置或升级付费计划

### Q2: 地图加载缓慢或经常卡顿
**原因**：Leaflet 库、卫星地图切片或 API 响应缓慢
**解决**：
- 在 ControlPanel 中切换到"暗黑"底图 (加载更轻量)
- 关闭气象雷达图层
- 清理浏览器缓存与 Cookie
- 尝试从另一个浏览器或设备访问

### Q3: 搜索功能无法找到某些地点
**原因**：OSM Nominatim 数据库可能未包含该地点
**解决**：
- 尝试使用更通用的地名 (如省份/城市而非小镇)
- 尝试英文名称搜索
- 如果仍不可用，直接输入经纬度坐标 (格式：`35.6895, 139.6917`)

### Q4: 本地存储的自定义点位突然消失
**原因**：浏览器 localStorage 可能被清空或超出配额
**解决**：
- 定期导出 CSV 备份自定义数据
- 检查浏览器配额使用情况 (DevTools → Storage)
- 在无痕模式下访问时，数据不会被保存

### Q5: 某个 Tab 组件显示空白或异常
**原因**：React 组件加载失败或数据源不可用
**解决**：
- 按 F12 打开控制台查看错误信息
- 尝试刷新页面 (Ctrl+R 或 Cmd+R)
- 尝试在另一个浏览器标签页中打开
