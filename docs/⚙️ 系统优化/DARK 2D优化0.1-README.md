📡 缺口一：交互与索敌系统 (Interactive UI)
在 Studio 里捏的金色站牌，目前只是一堆只有颜色没有灵魂的像素点。

缺什么：地图引擎不知道要点它。

前端要补的：

事件拦截网：在地图上铺一层不可见的“监听网”，当鼠标悬停或点击那些金点（transit-stop-label 图层）时，鼠标要自动变成“十字准星”或小手。

情报提取器：点击瞬间，代码要能像探针一样，刺透图层，把里面的 name（站名）和经纬度抓取出来。

战术弹窗 (Popup)：抓到站名后，在前端动态生成一个带有深空灰底色、琥珀金边框的赛博朋克风悬浮窗，实现“点击后才显影”的需求。

🌀 缺口二：动态雷达动画 (Dynamic Animations)
Mapbox Studio 渲染的是死物，“扩散环”和“呼吸灯”效果它做不到。

缺什么：时间轴和帧率控制。

前端要补的：

** CSS3 脉冲波**：在前端写一套 @keyframes 动画，当锁定某个站点时，在它的坐标上叠加一个透明的 HTML 节点，让它无限放大并变淡，模拟雷达波纹扩散。

(进阶) WebGL 扫图雷达：利用 requestAnimationFrame 让地图的某块区域或者整条铁路线呈现“流光”效果（像能量在血管里流动）。

✈️ 缺口三：空天一体化航线 (The Aerospace Engine)
这是的全球战略宏图——有些国家没铁路，得靠航线。

缺什么：Mapbox 底层数据库里根本没有天上的虚拟航线数据。

前端要补的：

大圆航线算法 (Great Circle)：引入 Turf.js 这种地理数学库。只要你给它起点（比如洛杉矶）和终点（东京）的坐标，它能在前端实时计算出一条贴合地球弧度的弯曲航线。

动态渲染层：把这条算出来的弧线，用代码作为“临时 GeoJSON 图层”强行画在这张暗黑底图的空中。


前端要补的：

Vercel 限流锁：必须把你的 Mapbox Token（尤其是如果用了 Secret Token）和地图加载动作，挂载到后端的防刷机制和前端的防抖（Debounce）函数上，死死护住那 5 万次的免费额度。
