# 🚀 WebGIS 终端 - v2.0 快速启动指南

> 日本旅行全息终端 · 卫星地形图系统

---

## 📋 项目概览

**当前版本：v2.0** | **框架：React 19 + Vite 7.3.1** | **样式：Tailwind CSS 4.2**

本项目是一个基于 WebGIS 技术的日本全境旅行漫游系统，集成了卫星地图导览、气象雷达覆盖、铁道网线追踪、实体坐标定位、次元圣地巡礼、跨国航线查询与智能摄影评分等功能。

## 🎯 5 分钟快速开始

### **前置要求**
- Node.js 18+ 版本
- npm 或 yarn 包管理器
- Git（可选，用于版本管理）

### **第 1 步：进入项目目录**

```bash
cd c:\Users\张\Desktop\github仓库\终端
```

### **第 2 步：安装依赖**

```bash
npm install
```

**预期输出：**
```
added 176 packages in 45s
```

> 💡 如果遇到 peer dependency 警告，不用理会。`.npmrc` 已配置 `legacy-peer-deps=true`。

### **第 3 步：启动开发服务器**

```bash
npm run dev
```

**预期输出：**
```
VITE v7.3.1  ready in 320 ms
➜  Local:   http://localhost:5174/
➜  press h to show help
```

浏览器**自动打开** → 看到 WebGIS 地图界面 ✅

## 🌐 环境变量配置

项目使用以下 API，需要配置环境变量：


在 Vercel Dashboard 中设置环境变量：
1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 选择项目
3. Settings → Environment Variables
4. 添加上述两个密钥

## 🐛 常见问题排查

### **Q1: 启动后白屏或加载缓慢**

**原因：** 可能需要 API 时间或地图瓦片加载中

**解决：**
```bash
# 等待 10 秒
# 按 F12 查看 Network 标签
# 确认 API 请求成功（状态 200）
```

### **Q2: "Cannot find module" 错误**

**原因：** 依赖未完全安装

**解决：**
```bash
# 清理缓存后重装
rm -r node_modules package-lock.json
npm install
npm run dev
```

### **Q3: 环境变量未生效**

**原因：** 需要重启开发服务器

**解决：**
```bash
# 停止开发服务器（Ctrl+C）
# 重新启动
npm run dev
```

### **Q4: Vite 端口被占用**

**原因：** 其他应用占用 5174 端口

**解决：**
```bash
# 指定不同的端口
npm run dev -- --port 3000
```
