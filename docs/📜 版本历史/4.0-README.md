# 🚂 地球终端系统 · EarthTerminal v4.0

**Japanese Railway Holographic Terminal · WebGIS Base Map System**

> 在 v3.1 **决定性瞬间三层架构**（数据网关 → 规则转换 → 智能匹配）之上，v4.0 将 **服务端数据平面** 与 **全球环境信号**（物候、极光）正式并入主线，并补齐 **可观测性与本地/生产双模运行** 说明。

**v4.0 核心升级**：🛰️ **Vercel Serverless 数据网关** | 🌸 **Open-Meteo 物候演算** | 🌌 **NOAA 极光 / 地磁规则联动** | 📡 **Sentry 监控** | 🧩 **赛博宽面板（Cyber Panel）交互**

---

## 📌 与 v3.1 文档的主要差异（为何升到 4.0）

| 维度 | v3.1 文档表述 | 当前代码事实 |
|------|----------------|--------------|
| 数据入口 | 以浏览器直连 WeatherAPI + Overpass + SunCalc 为主叙事 | **生产环境**气象与积温走 **`/api/weather`**，秘钥在服务端；物候、极光走 **`/api/phenology`**、**`/api/aurora`** |
| 摄影引擎 | 强调 UI 仅保留「三行魔法」 | **`photoEngine.js`** 仍为三层架构入口，但 **全息雷达评分 UI**、极光加成、**SunCalc 全量时刻** 等已扩展，不再是「仅三行」 |
| 规则库地磁 | `requiresGeomagneticActivity` 长期为占位 | **`ruleDataConverter`** 根据 **`gatewayData.aurora`**（NOAA OVATION 格点）写入 **`requiresGeomagneticActivity`**，极光类决定性瞬间可参与匹配 |
| 可观测性 | 未记载 | **`@sentry/react`** 在 **`main.jsx`** 初始化 |
| 目录结构 | 未列出 `api/` 与若干 utils | 需单独列出 **`api/*.js`**、**`cyberPanel.js`** 等 |

因此用 **v4.0 README** 统一描述「架构 + 部署 + 数据流」，避免 v3.1 文档与仓库脱节。

## 🛰️ Serverless 路由一览

| 路径 | 作用 | 上游数据 |
|------|------|-----------|
| **`GET /api/weather`** | 当前天气（瘦身字段）+ 可选积温相关 | WeatherAPI、`archive-api.open-meteo.com` |
| **`GET /api/phenology`** | 樱花 / 红叶等物候信号 | Open-Meteo Archive |
| **`GET /api/aurora`** | 格点极光强度（高纬） | NOAA `ovation_aurora_latest.json` |
| **`GET /api/astronomy`** | （保留）可与天文类扩展对接 | 视实现而定 |

**本地开发**：仅 `npm run dev` 时，浏览器访问的 origin 无 Node API，`/api/*` 需 **`npm run dev:stack`（`vercel dev`）** 或为 Vite 配置 **proxy**。详见仓库内 **`docs/ENV_AND_VERCEL_GUIDE.md`**。

**生产部署**：`vercel.json` 中 `api/**/*.js` 使用 `@vercel/node`；前端 **`npm run build`** 输出 `dist`。

## 🎬 智能摄影：v4.0 行为要点

1. **网关**：`fetchGlobalEnvironmentData(lat, lon)` 合并天文（SunCalc）、气候季节、**weather**、地形 POI、**ecology（物候）**、**aurora**。
2. **转换**：`convertToRuleFormat` 产出规则库所需字段；其中 **地磁/极光** 由 **OVATION 格点强度** 推导（与 UI 雷达使用同一量级阈值共识，当前代码约定 **≥ 8** 视为「有意义信号」；OVATION 实况典型量级约 0–25，随太阳活动变化）。
3. **匹配**：`getTopSuggestions` 驱动 **决定性瞬间** 文案；极光类规则（如 `requiresGeomagneticActivity: true`）在信号达标且其它条件满足时可命中。
4. **全息雷达 UI**：`photoEngine.js` 中在三层调用之外，保留 **S/A/B/C 评级**、标签条、**极光夜间加权**、黄金/蓝调等快门友好逻辑；标记 Popup 内按钮文案为 **「启动全息雷达扫描」**。

## 🧩 地图与交互补充

- **ControlPanel**：搜索、测距、图层与环境开关仍由 **`ControlPanel.jsx`** 承载。
- **Cyber Panel**：**`cyberPanel.js`** 提供宽屏信息浮层，**`useMapTools` / `useMapLayers`** 与之联动（打开/关闭、删除自定义点与面板协同）。
- **气象雷达**：全球枢纽与 OWM 相关配置仍在 **`mapConstants.js`**（如 `GLOBAL_WEATHER_NODES`、`OWM_API_KEY`）；与 **WeatherAPI 决策链** 并行存在，职责分离。

## 🛠️ 技术栈（与 v3.1 对齐处 + 新增）

- **React 19**、**Vite 7**、**Tailwind 4**、**SunCalc**、**Lucide React**

### 构建与脚本

- `npm run dev` — 仅前端
- `npm run dev:stack` — **`vercel dev`**，联调 `/api/*`
