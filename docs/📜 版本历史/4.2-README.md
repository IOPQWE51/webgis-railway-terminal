## 📌 v4.2 重大特性总结

### 🎯 数据闭环：战术收藏入库系统

**核心突破**：打通地图探索与星标库的数据流，实现真正的"所见即所得"数据闭环。

#### 功能特性

```javascript
// 全局收藏信号接收
window.__saveToCustomPoints = (name, lat, lon, category, btnElement) => {
    const newPt = {
        id: `custom_${Date.now()}`,
        name: name,
        lat: parseFloat(lat),
        lon: parseFloat(lon),
        category: category,
        source: '雷达手动捕获'
    };

    // 触发更新，App.jsx 会自动帮你存进 localStorage！
    if (onPointsUpdate) {
        onPointsUpdate(prev => [...prev, newPt]);
    }

    // 极客视觉反馈：变绿锁定
    if (btnElement) {
        btnElement.innerHTML = '✅ 编入成功';
        btnElement.style.background = '#10b981'; // 变成翡翠绿
        btnElement.style.boxShadow = '0 0 15px rgba(16, 185, 129, 0.4)';
        btnElement.style.pointerEvents = 'none'; // 防止连点
    }
};
```

**交互流程**：
1. 用户在地图上通过 **全息雷达扫描** 发现优质拍摄点
2. 点击 Popup 中的 **"⭐ 收藏到星标库"** 按钮
3. 系统自动将该点编入自定义星标库，同步存储至 `localStorage`
4. **视觉反馈**：按钮立即变绿并显示"✅ 编入成功"，提供即时操作确认

**技术亮点**：
- **响应式数据流**：利用 React 的 `onPointsUpdate` 回调，确保数据变更自动触发 UI 更新
- **本地持久化**：所有收藏数据自动同步至浏览器本地存储，刷新页面不丢失
- **极客视觉反馈**：采用翡翠绿高亮 + 发光阴影，提供明确的操作确认

### 🎨 战术准星：黑金重装版

**问题诊断**：v4.2.0 之前的赛博青/极光紫准星在浅色地图（卫星/地形模式）下 **对比度不足**，导致准星几乎不可见。

**解决方案**：全面重构为 **黑金高对比度配色**，确保在所有地图模式下都保持完美可见性。

#### 配色对比

| 版本 | 外框颜色 | 十字颜色 | 适用场景 |
|------|---------|----------|----------|
| **v4.2.0 之前** | 赛博青 `#22d3ee` | 极光紫 `#a855f7` | 仅深色地图 |
| **v4.2 及之后** | 深空灰 `#1e293b` | 琥珀金 `#fbbf24` | 全地图模式 ✅ |

**对比度测试**：
- **卫星图**（明亮）：黑金准星 WCAG AAA 级对比度
- **地形图**（中等）：琥珀金十字线清晰可见
- **暗黑模式**（深色）：深空灰外框提供柔和边界

### 📡 智能雷达：三引擎混合检索

**核心升级**：从单一 OSM Nominatim 引擎升级为 **三引擎混合雷达**，大幅提升地理编码的成功率和准确性。

#### 引擎配置

```javascript
// 🚀 引擎 A：Photon (基于 ElasticSearch，对中日英多语言抗干扰极强)
const photonUrl = `https://photon.komoot.io/api/?q=${encodeURIComponent(smartQuery)}&limit=1`;

// 🎯 引擎 B：ArcGIS (加上英文后缀后，绝对不会再定位到济南小卖部)
const arcGisUrl = `https://geocode.arcgis.com/arcgis/rest/services/World/GeocodeServer/findAddressCandidates?f=json&singleLine=${encodeURIComponent(smartQuery)}&maxLocations=1`;

// ♻️ 引擎 C：OSM Nominatim (强制加入 ja 日语 header 偏好，防止富冈/福冈惨案)
const osmUrl = `https://nominatim.openstreetmap.org/search?q=${encodeURIComponent(placeName)}&format=json&limit=1&accept-language=zh-CN,ja,en-US`;
```

**语义增强系统**：
```javascript
// 🧠 语义增强 (Semantic Sniffing)
// 强行给纯中文地标加上英文后缀，防止商业引擎将其误判为国内的同名小公司
let smartQuery = placeName;
if (placeName.includes('机场') || placeName.includes('空港')) smartQuery += ' Airport';
if (placeName.includes('站') || placeName.includes('駅')) smartQuery += ' Station';
if (placeName.includes('旅舍') || placeName.includes('酒店')) smartQuery += ' Hotel';
```

**容错机制**：
- **Photon**：主引擎，抗干扰能力强
- **ArcGIS**：企业级后备，适合复杂地址
- **OSM**：最终兜底，确保永不失败

### 🌐 跨端一致性：面板透明度同步

**问题诊断**：v4.2.2 发现生产环境面板透明度与本地开发环境不一致，导致 **跨端视觉差异**。

**解决方案**：强制提升背景不透明度到 `0.95`，并使用 `!important` 锁定样式。

#### CSS 修复

```css
/* 🚀 核心修复：提升背景不透明度到 0.95，并使用 !important 锁定 */
.cyber-panel.cyber-panel--map-dock {
    background: rgba(255, 255, 255, 0.95) !important;

    /* 强化模糊感，确保在任何背景下都有沉浸式磨砂感 */
    backdrop-filter: blur(25px) saturate(180%);
    -webkit-backdrop-filter: blur(25px) saturate(180%);

    /* 左侧高光边保持不变 */
    border-left: 1px solid rgba(255, 255, 255, 0.5);
}
```

**技术亮点**：
- **!important 锁定**：防止第三方样式覆盖
- **双重滤镜**：`blur(25px)` + `saturate(180%)` 打造极致毛玻璃效果
- **z-index 提升**：从 `30` 提升至 `1200`，确保面板始终在顶层

### 🗺️ 渲染优化：Leaflet 瓦片亚像素缝隙修复

**问题诊断**：在高缩放级别下，Leaflet 瓦片之间出现 **1 像素左右的缝隙**，严重影响地图美观度。

**解决方案**：使用现代独立缩放法，让每张瓦片边缘向外扩张约 1.2 像素，完美互相覆盖。

#### CSS 修复

```css
/* 🚀 终极消除 Leaflet 瓦片亚像素缝隙 (现代独立缩放法) */
.leaflet-tile-container img.leaflet-tile {
    /* 使用独立 scale 属性，绝对不会破坏 Leaflet 的底层坐标定位！ */
    /* 放大 1%，让每张瓦片边缘向外扩张约 1.2 像素，完美互相覆盖 */
    scale: 1.01 !important;
}
```

**技术亮点**：
- **独立 scale 属性**：不影响 Leaflet 的底层坐标定位系统
- **1.01 倍微调**：刚好覆盖缝隙，不过度放大导致瓦片重叠
- **跨浏览器兼容**：使用标准 CSS scale 属性，支持所有现代浏览器
