# 🌍 EarthTerminal v4.2.1

**地球终端系统 · Earth Terminal System**

> 一个专为日本深度旅行者打造的全息战术指挥终端。本系统集成了高精度卫星地形图、铁道网线追踪、智能摄影评分、气象雷达覆盖、跨国航线查询等核心功能，为用户提供一站式的旅行规划与决策支持平台。

**版本号：v4.2.1** | **最后更新：2026-03-31**

---

## 📋 项目概述

### 核心定位

本项目是一个基于 WebGIS 技术的**日本全境旅行战术终端**，专注于为自由行旅客提供：

- 🗺️ **高精度地形导航** - 基于 Esri 卫星地图的全日本地图系统
- 🚃 **铁道网线追踪** - 青春18车票全日本50站骨架网络
- 📸 **智能摄影评分** - V4.0 全息雷达摄影引擎，100+ 决定性瞬间智能预警
- 🌦️ **气象雷达覆盖** - 实时降雨/云层/极光等多维环境数据
- 🎯 **战术收藏系统** - 地图探索与星标库的数据闭环
- 🌐 **跨国航线查询** - 中日韩主要城市航线雷达
- 💱 **汇率物价指数** - 7种核心货币实时汇率与CPI物价指数

### 技术架构

**前端技术栈：**
- **React 19.2.0** - 最新版React框架
- **Vite 7.3.1** - 极速开发构建工具
- **Tailwind CSS 4.2.1** - 原子化CSS框架

**部署平台：**
- **Vercel Serverless** - 全球边缘计算部署

## 🎯 核心功能模块

### 1. 高精度地形终端 (MapEngine)

**功能描述：**
全日本范围的高精度卫星地形图系统，支持多种地图模式切换与智能点位管理。

**核心特性：**

#### 1.1 多层地图叠加
- **底图模式**：Esri 卫星图 / Esri 地形图 / CartoDB 暗黑图 / CartoDB 极简图
- **天气图层**：降雨雷达 / 云层覆盖 / 极光概率（NOAA OVATION）
- **点位聚合**：自动聚合相近点位，支持智能缩放

#### 1.2 战术准星系统
```javascript
// 黑金重装战术准星 - 解决浅色地图不可见问题
- 深空灰外框 (#1e293b) - 高对比度边界
- 琥珀金十字线 (#fbbf24) - 确保全地图模式可见
- 同心圆系统 - 提升战术瞄准感
```

#### 1.3 智能点位管理
- **预设点位**：50个青春18铁道骨架站点
- **自定义点位**：支持用户添加收藏地标
- **分类筛选**：车站 / 机场 / 动漫圣地 / 旅舍 / 景点
- **一键入库**：地图探索点位可直接编入星标库

#### 1.4 测距系统
- **测距模式**：点击地图任意两点进行测距
- **实时显示**：距离数值实时更新（公里）
- **清除连线**：不清除已测量点位，只清除连线
- **退出测距**：完全退出测距模式

#### 1.5 三引擎智能雷达
```javascript
// 地理编码三大引擎
1. Photon API (主引擎) - 基于ElasticSearch，抗干扰能力强
2. ArcGIS (备用引擎) - 企业级地理编码，适合复杂地址
3. OSM Nominatim (兜底引擎) - 确保永不失败
```

**交互流程：**
1. 用户在地图上点击"雷达检索"按钮
2. 输入地名（支持中英文）
3. 系统自动调用三大引擎进行地理编码
4. 找到坐标后自动定位并添加标记

## 🐛 常见问题

### Q1: 启动后白屏或加载缓慢

**原因：** 地图瓦片加载中或 API 请求延迟

**解决：**
```bash
# 等待 10 秒
# 按 F12 查看 Network 标签
# 确认 API 请求成功（状态 200）
```

### Q2: "Cannot find module" 错误

**原因：** 依赖未完全安装

**解决：**
```bash
# 清理缓存后重装
rm -r node_modules package-lock.json
npm install
npm run dev
```

### Q3: 环境变量未生效

**原因：** 需要重启开发服务器

**解决：**
```bash
# 停止开发服务器（Ctrl+C）
# 重新启动
npm run dev
```

### Q4: Vite 端口被占用

**原因：** 其他应用占用 5174 端口

**解决：**
```bash
# 指定不同的端口
npm run dev -- --port 3000
```
