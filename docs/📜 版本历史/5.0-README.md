# 🗺️ 地球终端系统 · EarthTerminal v5.0

**Japanese Railway Holographic Terminal + Mapbox Dual-Map Architecture**

> 在 v4.0 的 Serverless 数据网关与全息雷达系统之上，v5.0 引入 **Mapbox GL JS** 双地图架构，实现 **Leaflet 航拍地景** 与 **Mapbox 2D 矢量** 的互补显示，新增 **Dark 2D 自定义样式** 与 **数据中心** 完整集成。

**v5.0 核心升级**：🎨 **Mapbox GL JS 集成** | 🌃 **Dark 2D 自定义样式** | 📊 **数据中心 CSV 批量导入** | ✈️ **跨国航线雷达** | 💱 **汇率引擎** | 🎆 **花火雷达** | ⛩️ **巡礼雷达**

---

## 📌 与 v4.0 的主要差异

| 维度 | v4.0 | v5.0 |
|------|------|------|
| 地图引擎 | 单一 Leaflet | **双架构**：Leaflet（航拍地景） + Mapbox（2D 矢量） |
| 地图样式 | Esri/Carto 卫星底图 | **Mapbox 自定义 Dark 2D** + 经典 Streets/Light/Dark |
| 数据管理 | 仅本地存储 | **数据中心**：CSV 批量导入 + Google 智能检索 |
| 功能模块 | 摄影引擎 + 规则系统 | **新增**：航线雷达、汇率引擎、花火雷达、巡礼雷达 |
| 组件架构 | 单一 MapEngine | **模块化引擎**：MapEngine + AviationEngine + ExchangeEngine 等 |

## 🎨 双地图系统详解

### 🛰️ Leaflet 地图 (MapEngine.jsx)

**用途**：高精度航拍地景、摄影辅助、全息雷达评分

```javascript
// 底图来源
- Esri World Imagery (卫星)
- CartoDB Positron (明亮)
- CartoDB Dark Matter (暗色)
- OpenStreetMap (标准)
```

**核心功能**：
- 🎯 **全息雷达评分**：S/A/B/C 评级系统
- 📸 **决定性瞬间**：黄金时刻、蓝调时刻、极光探测
- 🗺️ **航拍点管理**：自定义标记 + 预设基地点
- 🔍 **Cyber Panel**：宽屏信息浮层
- 📏 **测距工具**：点击测距、多边形面积

### 🌃 Mapbox 地图 (MapboxMapTactical.jsx)

**用途**：Dark 2D 矢量地图、城市铁路网络、数据可视化

```javascript
// 样式定义
- Dark 2D (自定义): mapbox://styles/iopqwe51/cmnoq0jyc008501sg88f6en5z
- Mapbox Dark: mapbox://styles/mapbox/dark-v11
- Mapbox Light: mapbox://styles/mapbox/light-v11
- Streets: mapbox://styles/mapbox/streets-v12
```

**核心功能**：
- 🌃 **自定义 Dark 2D**：优化的暗色矢量地图
- 🚇 **地铁网络显示**：transit、station 图层
- 📍 **标记点系统**：GeoJSON 图层渲染
- 🏙️ **城市预设**：东京、大阪、纽约等
- 🎨 **样式实时切换**：无需重新加载

## 🚀 快速开始

### 环境要求

- Node.js 18+
- npm 或 yarn

### 安装依赖

```bash
npm install
```

### 配置环境变量

复制 `.env.example` 为 `.env` 并填入 API Keys：

```bash
# 必需：Mapbox Public Token (格式: pk.eyJ1...)
VITE_MAPBOX_ACCESS_TOKEN=你的_Mapbox_Public_Token

# 可选：Google Maps API
VITE_GOOGLE_MAPS_API_KEY=你的_Google_API_Key

# 可选：天气 API
VITE_WEATHER_API_KEY=你的_天气_API_Key
```

**重要**：
- Mapbox Token 获取：https://account.mapbox.com/ → Access tokens → Default public token
- ⚠️ 使用 **Public Token**（pk. 开头），不要使用 Secret Token（sk. 开头）

### 启动开发服务器

**方法1：使用启动脚本（Windows）**
```bash
# 双击
终端.bat
```

**方法2：命令行启动**
```bash
# 仅前端
npm run dev

# 前端 + Serverless API (需要 Vercel CLI)
npm run dev:stack
```

访问地址：
- 本地：http://localhost:5173
- 网络：http://192.168.x.x:5173

### 构建生产版本

```bash
npm run build
npm run preview
```

## 🎯 使用指南

### 添加自定义标记点

**方法1：地图点击**
1. 打开 **高精度地形终端** 或 **Dark 2D 地图**
2. 点击地图任意位置
3. 自动生成标记点

**方法2：CSV 批量导入**
1. 切换到 **数据解析与管理** 标签
2. 准备 CSV 文件（格式见上文）
3. 拖拽文件到导入区域
4. 点击 **导入数据**

**方法3：Google 智能检索**
1. 在数据中心输入地标名称（如"东京塔"）
2. 点击 **智能检索**
3. 自动获取坐标并添加

### 切换地图样式

**在 Dark 2D 地图中**：
1. 点击样式按钮
2. 选择：
   - **Dark 2D (自)** - 自定义暗色矢量
   - **Mapbox Dark** - 官方暗色
   - **Light** - 明亮主题
   - **Streets** - 经典街道

### 城市快速切换

**Dark 2D 地图支持**：
- 🗼 **东京**：霓虹都市 · 交通枢纽
- 🏰 **大阪**：关西门户 · 美食之都
- 🗽 **纽约**：世界十字路口 · 大苹果

### 航线规划（跨国航线雷达）

1. 切换到 **跨国航线雷达** 标签
2. 在地图上标记起点和终点
3. 调用航线绘制 API：
```javascript
// 控制台执行

  { lng: 139.6503, lat: 35.6762 }, // 东京

```

## 🐛 故障排除

### 地图不显示

**症状**：只看到文字，没有地图

**解决方案**：
1. 检查 `.env` 中的 `VITE_MAPBOX_ACCESS_TOKEN` 是否正确
2. 确认使用的是 **Public Token**（pk. 开头）
3. 打开浏览器控制台（F12）查看错误信息
4. 硬刷新页面（Ctrl + Shift + R）

### Token 权限错误

**错误信息**：`Not authorized to access styles`

**解决方案**：
1. 登录 Mapbox Account
2. 检查 Token 的 **Scopes** 是否包含：
   - ✅ styles:tiles
   - ✅ fonts:read
   - ✅ styles:read
3. 如果没有，重新创建 Token

### 样式无法加载

**症状**：地图空白，控制台报错 404

**解决方案**：
1. 确认样式 URL 格式正确：
   ```
   mapbox://styles/用户名/样式ID
   ```
2. 检查样式是否已在 Mapbox Studio **发布**
3. 验证用户名是否正确

## 🔐 安全说明

### API Key 管理

**前端使用（VITE_ 前缀）**：
- ✅ Mapbox Public Token
- ✅ Google Maps API Key（受限）
- ⚠️ 这些 Key 会暴露在浏览器中

**后端使用（无 VITE_ 前缀）**：
- 🔒 Weather API Secret Key
- 🔒 NASA API Key
- 🔒 Astro App Secret
- ✅ 这些 Key 仅在 Serverless 环境中使用

### 环境变量隔离

**开发环境** (`.env`)：
```bash
VITE_MAPBOX_ACCESS_TOKEN=pk.xxx  # 前端可见
WEATHER_API_KEY=xxx              # 后端专用
```

**生产环境** (Vercel Dashboard)：
- 前端 Key：在 **Environment Variables** 中设置（无 VITE_ 前缀）
- 后端 Key：在 **Serverless Function** 环境变量中设置
